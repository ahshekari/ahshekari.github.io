📈 Time Series Forecasting – Transactions

Exploring Trends, Seasonality, and Predictive Modeling (2019–2021)

← Back to Home

Project Overview

This project analyzed historical transaction data from 2019–2021 to identify patterns and forecast future transaction amounts. Using Python, Prophet, and XGBoost, we explored seasonality, trends, and volatility to build robust predictive models for Q1 2021.

Objectives

  • Understand daily, weekly, and monthly transaction patterns.
  • Decompose the time series into trend, seasonality, and residuals.
  • Build and compare forecasting models using Prophet and XGBoost.
  • Incorporate realistic volatility adjustments for more accurate predictions.

Tools & Methods

  • Python Libraries: Pandas, NumPy, Matplotlib, Seaborn, Statsmodels
  • Models: Prophet (trend & seasonality), XGBoost (gradient boosting)
  • Evaluation Metrics: RMSE, MAE, R², MAPE, Direction Accuracy
  • Data Engineering: Lag features, rolling means, time-based features

Key Findings

  • Prophet Results: RMSE ≈ 21,645 | MAE ≈ 15,782 | R² = 0.13 | Direction Accuracy ≈ 62%
  • XGBoost Results: RMSE ≈ 13,325 | MAE ≈ 8,202 | R² = 0.67 | Direction Accuracy ≈ 86%
  • XGBoost significantly outperformed Prophet in predictive accuracy and trend direction.
  • Separating credit vs. debit transactions improved forecasting precision by capturing distinct patterns.
  • Adding volatility adjustments made forecasts align more closely with real-world fluctuations.

Impact

The project demonstrated how time series forecasting can be applied to financial transaction data, providing insights into seasonality, transaction spikes, and forward-looking projections. It highlighted the strengths of combining machine learning (XGBoost) with statistical forecasting (Prophet).
